Command Line

cat: the cat command can list the contents of files to the terminal window.
Syntax:
cat [-AbeEnstTuv] [--help] [--version] fileName
-A, --show-all equivalent to -vET
-b, --number-nonblank number nonblank output lines
-E, --show-ends display $ at end of each line
-n, --number number all output lines

chmod: change access permissions, change mode.
Syntax:
chmod [-cfvR] [--help] [--version] mode file...
User letter
The user who owns it u
Other users in the file's Group g
Other users not in the file's group o
All users a

Permission letter
Read r
Write w
Execute x

mv: move or rename files or directories.
Syntax:
mv [options]... Source Dest
mv [options]... Source... Directory
-b Make a backup of each file that would otherwise be overwritten or removed.
-f Remove existing destination files and never prompt the user.
-i Prompt whether to overwrite each existing destination file regardless of its permissions.

rm: remove files (delete/unlink).
Syntax:
rm [options] name...
-i Prompt before every removal.
-r Remove directories and their contents recursively.
-f Ignore nonexistent files, never prompt.

echo: display message on screen.
Syntax:
echo [options]... [String]...
-e Enable interpretation of the following backslash-escaped characters in each String:
\n New line
\c Suppress trailing newline
\t Horizontal tab
\v Vertical tab

unzip: list, test and extract compressed files in a ZIP archive.
Syntax:
unzip [-Z] [-cflptTuvz[abjnoqsCDKLMUVWX$/:^]] file[.zip] [file(s) ...] [-x xfile(s) ...] [-d exdir]

Example: $ unzip lab1resources.zip


cp: copy one or more files to another location.
Syntax:
cp [options] source dest
cp [options] source... directory
-a same as -dpR
-d preserve links
-f remove existing destinations, never prompt
-i prompt before overwrite
-p preserve file attributes if possible
-r copy recursively

Example: $ cp test newtest

File, Directories and Permissions

touch: change file timestamps, change the access and/or modification times of the specified files.
Syntax:
touch [options]... File...

Example: $ touch sample.txt

Example: $ touch {1..10}.txt


mkdir: create new folder(s), if they do not already exist.
Syntax:
mkdir [Options] folder...

Example: $ mkdir cs133

Example: $ mkdir -p warwick/cs133


cd: change the current working directory to a specific folder.
Syntax:
cd [dirName]
~ home     . or ./ current directory     cd .. move up one folder  

pwd: print working directory.
Syntax:
pwd [--help][--version]

ls: list information about files..
Syntax:
ls [Options]... [File]...
-a list all entries including those starting with a dot.
-l list details
-t sort by modification time
-s print size of each file, in blocks

chmod: change access permissions, change mode.
Syntax:
The operator '+' causes the permissions selected to be added to the existing permissions of each file; '-' causes them to be removed;
u: the user who owns it
g: other users in the file's Group
o: other users in the file's Group
a: all users

Example1: allow write permission to everyone
$ chmod a+w file

Example2: deny execute permission to everyone
$ chmod a-x file

Example3: make a file readable and writable by the group and other
$ chmod go+rw file

Setting Permissions Shorthand

This process involves: representing a permission string using binary, where 1 and 0 are respectively used to grant and restrict permission; and then representing the binary using octal.
  • rwx --- --- = 111 000 000 = 700
  • rw- rw- rw- = 110 110 110 = 666
  • rwx rwx rwx = 111 111 111 = 777
Example: allow all permission to everyone
$ chmod 777 file
